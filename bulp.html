Day #2 - allocators

To parse expressions I needed allocators, due to the recursive nature of AST's.
An AST node could just be a leaf (primary expression), or non-leaf (idk how to call them), eg. binary expressions.
So I went to the Odin docs and tried to search for an arena or bump allocator, Since I'm so used to using them for parsing recursive structues. I was pleased to find an arena. From what I understand currently there are TWO arenas, one with dynamic backing memory, and static. Concerned with simplicity I went with the statically backed one. Other than that it was pretty easy. One annoying but not deal-breaking thing. Allocations return a rawptr, and an error, so I HAVE to define a new binding for the ptr which will have the AstNode type associated with it. Since I have only a single operator: `+`, instead of going the easy route, I immediately laid out the foundation for operator precedence. Since I implemented this in the morning, tests will come later in the day.
