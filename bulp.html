Day #1 - beginning of the bulp

I feel like Odin might be a better choice for a compiler. Safer enums, more hand-holding memory management, testing built-in. So yes I'm going with Odin, all of this might just be peak delusion that a new language will fix my problems, but I truly believe in Odin for begin my savior. One last thing. This is my first project with Odin.
Rant over.
I began the day by going over the Odin Overview, and first impressions? I like it. It's basic, but not as barebones as Golang. Actual enums instead of the lazy actor that's in golang (yes I'll call it golang). Optionals (in the form of Maybe) even tho the documentation suggests using multiple return arguments instead of this type is a decent addition.
The standard library seems cool :D
The fact that the majority(?) of libc is available is nice, I'll need to see how easy it is to integrate C code though.
Two last remarks
The using statement is a cool addition that makes refactoring code easier.
I don't have code examples for this, but it's easy to write concise basic code in Odin
